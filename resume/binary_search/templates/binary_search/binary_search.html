Cуть алгоритма бинарного поиска это поиск элементов за логарифмическое время.

C увеличением элементов бинарный поиск будет работать еще быстрее по сравнению с простым поиском, в котором время выполнения поиска является линейным. Это экономит время и ресурс самого проекта и клиента.

log8 == 3, для списка из 8 элементов не более 3 проверок.
log1024 == 10, для списка из 1024 элементов не более 10 проверок.
